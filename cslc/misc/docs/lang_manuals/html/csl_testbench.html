1.2 CSL Testbench Commands

1.2.1 CSL testbench

csl_testbench testbench_object_name;
Description :

Testbench_declaration. Creates a testbench object named testbench_object_name.

[ CSL Testbench Commands Summary ]

Example :

In this example, a csl testbench named tb is created. A testbench is a container.

FIGURE 1.1 A testbench tb


CSL code
//declare a testbench named "tb"
csl_testbench tb;
Verilog code
module tb;
endmodule
set_testbench_verilog_filename(name);
Description :

The default name of the Verilog module and the filename for the testbench is the name of the csl_testbench instance. Set_testbench_verilog_filename(name) will override the default testbench name with name.

Use csl_include to add an include file containing Verilog code to the Verilog testbench.

[ CSL Testbench Commands Summary ]

Example :

//

CSL code
//
Verilog code
//
add_logic(simulation_timeout_counter, numeric_expresion);
Description :

The testbench will stop the simulation if it reaches time numeric_expression. This will add a counter which if it is still running at numeric_expression will stop the simulation and print an error message stating that the simulation did not finish by time numeric_expression.

FIGURE 1.2


CSL code
csl_testbench tb_a{
tb_a(){
set_simulation_timeout_count(1000);
}};
csl_testbench tb_b{
tb_a tb_a;
tb_b(){
set_simulation_timeout_count(tb_a.get_simulation_timeout_count());
}};
Verilog code
module tb_a();
parameter MAX_SIM_TIMEOUT=1000;
endmodule
module tb_b();
parameter MAX_SIM_TIMEOUT=1000;
endmodule
